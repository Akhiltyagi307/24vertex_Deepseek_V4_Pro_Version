# 24Vertex — Platform Foundations (shared across all portals)

**Snapshot:** 2026-06-12 · branch `main` @ `27d7fbb`. Verified against current code on this date.
**Product:** 24Vertex (display) · `vertex24` (technical slug) · `eduai` (legacy slug). NCERT curriculum, grades 6–12, India.
**Purpose:** the cross-cutting systems every portal depends on. The four portal docs ([student](student-portal.md), [teacher](teacher-portal.md), [parent](parent-portal.md), [admin](admin-portal.md)) reference this file instead of repeating it. **[Plain]** = any reader · **[Technical]** = dense, for engineers/LLMs.

## 1. Stack — [Technical]
- **Framework:** Next.js 16 (App Router, React 19, Server Components + Server Actions), TypeScript, on Vercel (Node 24).
- **Data:** Supabase Postgres via **Drizzle ORM** (`src/db/schema/*.ts`, typed server queries) and the **Supabase JS client** (`@supabase/ssr`, RLS-scoped reads + Realtime). Storage buckets for attachments + PDFs.
- **AI:** Vercel AI SDK v6 (`ai`, `@ai-sdk/deepseek`, `@ai-sdk/openai`, `@ai-sdk/react`) behind a model router — DeepSeek primary (cost), OpenAI fallback (capability).
- **UI:** Tailwind v4 + shadcn/Radix; charts Recharts + Plotly + Mafs + `function-plot`; math KaTeX (`rehype-katex`/`remark-math`); PDF `@react-pdf/renderer`; rich text TipTap; chemistry `smiles-drawer`; diagrams Mermaid/Three; OCR `tesseract.js`.
- **Infra:** Sentry (`instrumentation.ts`, `sentry.*.config.ts`; PII scrubber `src/lib/sentry/before-send.ts` runs on events AND the Logs stream in all 3 runtimes); **Postgres-backed rate limiting** (`src/lib/ratelimit/**` — fixed-window `rlConsume` over a dedicated `ratelimitSql` client, circuit breaker + LRU deny cache; shared fail policy `fail-policy.ts` denies on degraded limiter in prod/preview). Upstash Redis exists only as a health-ping target (`run-all-health-pings.ts`), not the limiter backend. Resend email; Razorpay billing.
- **Background jobs:** `practice_jobs` worker (`app/api/internal/practice/run-jobs`, cron-secret-gated) claims via `practice_claim_jobs` RPC; reclaim leases off `claimed_at` (not progress-bumped `updated_at`), completion fenced on `claimed_by`; `operator_jobs` has lease/reclaim too; pg_cron stall detection via `app/api/internal/ops/cron-heartbeat` (queue-backlog probe).
- **Boot assertions (`instrumentation.ts`):** prod refuses to start on plaintext `ADMIN_PASSWORD`, unset SaaS enforcement, or a missing admin second factor (`ADMIN_IP_ALLOWLIST` and/or `ADMIN_TOTP_SECRET`).
- **Request proxy:** `proxy.ts` (Node runtime) runs per-portal gates, CSP nonce per path, maintenance routing, and request-id correlation on every request.

## 2. Roles & authentication — [Technical]
- **Students / parents / teachers:** Supabase Auth (email/password + recovery). The signed-in user maps to a `profiles` row with `role ∈ {student, parent, teacher}`.
- **User-auth rate limiting (`src/lib/auth/rate-limit.ts`):** app-level limits on login/signup/password-reset server actions (`app/(auth)/**/actions.ts`), on top of Supabase's coarse project limits. Account-first keying (SHA-256 of normalized email — never raw PII) so a shared school NAT IP isn't locked out: login 10/15min per email + 60/15min per IP; reset 5/h per email + 20/h per IP; signup 40/h per IP. Fail-closed in prod via `shouldDenyOnDegraded`.
- **Password recovery:** the post-recovery window cookie (`src/lib/auth/recovery-window.ts`, `edu_recovery_window`, 10min) is **HMAC-signed** when `RECOVERY_WINDOW_HMAC_SECRET` is set (`<userId>.<nonce>.<hmac>`, tamper-evident); unsigned legacy form otherwise. Defense-in-depth — the authoritative gate is the Supabase-session match in `updatePasswordAction`.
- **Admins:** a **separate hardened stack** (no Supabase Auth): bcrypt (`ADMIN_PASSWORD_HASH_B64`) + TOTP 2FA (`otplib`) + `jose` JWT with **`kid`-based key rotation** + IP allowlist + login rate limit + panic-revoke. Plaintext `ADMIN_PASSWORD` is **rejected in production**, and prod boot requires `ADMIN_IP_ALLOWLIST` and/or `ADMIN_TOTP_SECRET` (`instrumentation.ts`).
- **Admin TOTP is single-use:** `consumeAdminTotp()` (`src/lib/admin/auth.ts`) verifies the code AND consumes its 30s clock step via an `admin_runtime_kv` compare-and-set (`runtime-pg.ts`; fails open on DB error). Used at login and at every **step-up** site: impersonation (`/api/admin/users/[id]/impersonate` — fail-closed, fresh code required), panic, hard-delete, SQL run, compliance erase / retention run-now. `staff-override` writes its audit row via `writeAdminActionStrict` (action fails if the audit write fails).
- **Internal cron/worker auth (`src/lib/internal/cron-auth.ts`):** prod requires `Authorization: Bearer <CRON_SECRET>`; with no secret, the dev bypass only applies on loopback AND `ALLOW_UNAUTHENTICATED_CRON_DEV=1` + `NODE_ENV=development`. Header (password-grant) auth is refused on any Vercel deployment.
- **Guard helpers (`src/lib/auth/`):** `requireUser({ role })` dispatches to `requireVerifiedStudent`, `requireParent`, `getVerifiedTeacherSession`. Portal layouts use these (student → `requireVerifiedStudent()`, parent → `requireParent()`, teacher protected → `requireVerifiedTeacher()` with suspended/not-verified redirects).
- **CSRF/origin gating:** `proxy.ts` runs `studentProxyGate`, `parentProxyGate`, `teacherProxyGate`, `adminProxyGate`, `billingProxyGate`, `feedbackProxyGate`, `contactProxyGate`. Each rejects mismatched `Origin` on `POST/PUT/PATCH/DELETE` via `originAllowed` (`src/lib/security/origin-guard.ts`). Server-to-server (no Origin) passes through.

## 3. Data model — table backbone — [Technical]
*(Drizzle: `src/db/schema/`. Grouped by domain.)*
- **Identity/links:** `profiles` (role, grade/section/stream, `studentLinkCode`, `organizationId`, suspend/soft-delete, `onboarding_welcome_seen_at`), `parent_student_links` (pending → student-confirmed via `confirm_parent_link`/`reject_parent_link` RPCs; a trigger restricts what students can update), `teacher_student_links`, `teacher_organization_memberships`, `organizations`.
- **Curriculum:** `subjects` → `topics` (unit→chapter→topic + learning objectives), `topic_context_chunks` (RAG source).
- **Assessment/practice:** `tests` (`test_type ∈ {self, assigned, review}`; `client_request_id` idempotency key, unique per student), `questions` (+ pgvector `embedding`), `student_answers`, `test_reports`, `question_flags`, `performance_tracker` (+ SM-2-lite review-schedule columns `next_review_at`/`review_interval_days`/`review_ease`/`consecutive_good`, partial index `idx_perf_next_review`), `practice_jobs` (`job_type ∈ {grade, pdf, auto_submit, email, tracker_update, assign_generate_test, review_generate, student_generate_test}`; partial-unique active-job dedup indexes for the two generate types), `practice_generation_runs`/`_steps` (`request_mode` CHECK widened to include `assignment_worker`/`review_worker`), `practice_analytics_events`, `student_activity_streaks` (+ streak-freeze columns `bridged_weeks`/`freezes_available`/`freeze_last_used_week`).
- **Assignments:** `assignments`, `assignment_submissions`, `assignment_questions` (teacher-authored question template for manual assignments, RLS teacher-read; materialized per student by the no-LLM `practice_create_manual_assigned_test` RPC, service-role only).
- **Doubt (AI tutor):** `doubt_conversations`, `doubt_messages`, `doubt_message_attachments`.
- **Billing:** `plans`, `subscriptions` (+ `dunning_started_at` dedicated dunning clock, `UNIQUE(razorpay_subscription_id)`), `usage_periods`, `quota_grants`, `free_trial_claims`, `identity_blocklist`, `payments`, `coupons`, `coupon_redemptions`, `billing_events`, `billing_plan_changes`, `billing_reconciliation_drift`, `billing_action_failures`, `usage_notification_log`, `admin_refund_idempotency`.
- **Admin/ops:** `admin_sessions`, `admin_login_rate`, `admin_runtime_kv` (also TOTP single-use + session-revocation tombstones), `admin_saved_views`, `admin_action_log` (append-only), `feature_flags`, `operator_jobs` (`jobs`, now leased/reclaimable), `integrity_check_results`, `service_health_pings`.
- **AI governance:** `ai_prompts` (versioned), `ai_calls` (cost/tokens).
- **Teacher AI:** `teacher_class_insights` (per-teacher cache of AI class-insight narratives keyed by scope + `prompt_version`, validated by `data_fingerprint`).
- **Comms:** `broadcasts`, `email_templates`, `email_webhook_events`, `comms_audit`, `notifications` (defined in `comms-audit.ts`).
- **Compliance/safety:** `parental_consents`, `compliance_requests`, `retention_policies`, `moderation_flags`, `content_blacklist`, `teacher_approval_history`, `user_feedback_reports`.
- **SQL config functions** (flipped via `CREATE OR REPLACE`, no migration): `review_scheduler_enabled()` (kill switch), `review_scheduler_rollout_pct()` + `review_scheduler_cohort_org_ids()` (staged rollout), `streak_freeze_enabled()`.
- **pg_cron:** `practice-review-scheduler-nightly` (02:30 UTC → `POST /api/internal/practice/review-scheduler` with vault-stored cron secret), `teacher-class-insight-prune-weekly` (Sun 03:00 UTC, deletes insights unserved for 90 days).
- **RPC authz hardening** (`20260702000000_security_idor_hardening.sql`, `20260706000000_restore_parent_link_confirmation_and_harden_admin_rpcs.sql`): SECURITY DEFINER RPCs now enforce function-level authorization — `refresh_student_activity_streak` checks `auth.uid()` ownership, `practice_enqueue_job` lets an authenticated caller enqueue only for a test they own (workers via service_role), worker-only RPCs (`practice_claim_jobs`, `practice_generate_review_test`, `practice_generate_self_test`, `practice_create_manual_assigned_test`) check `auth.role() = 'service_role'`, teacher helpers (`teacher_can_access_student`, `teacher_filter_accessible_student_ids`, …) gate teacher reads, and `admin_set_teacher_verified` is postgres-only with a GUC defense-in-depth check.

## 4. AI pipelines — [Technical]
- **Practice generation** (`src/lib/practice/practice-generation-pipeline.ts`): config resolve → quota preflight → blueprint (deterministic or LLM) → evidence pack / RAG (`topic_context_chunks`) → batched generation (sister-brief for cross-batch coherence) → editor → repair + deterministic autofix + single-question replacement → quality gates (pgvector near-dup, topic concentration, visual/chunk alignment) → validation + audit/normalize + KaTeX normalize → moderation pre/post. Resilience: model router failover, reason-aware retries, low-context fallback; tail-latency guard (`runModelWithTailGuard`, 75s soft timeout + one retry) now wraps the default single-call path too, not just parallel batches. Streaming via NDJSON envelope (`generate-stream-envelope.ts`: `partial`/`done`/`error`) mapped to student-facing checklist buckets (`generation-progress-buckets.ts`, fed by the pipeline's `onStage` callback). Telemetry → `practice_generation_runs`/`_steps`.
- **Durable generation (flag-gated):** `PRACTICE_ASYNC_GENERATE` switches student "Generate" from one streaming HTTP request to a `student_generate_test` job — enqueue `POST /api/student/practice/generate`, poll `GET …/generate/status`, worker runs `materializeStudentGeneratedTest` (`src/lib/practice/student-generation.ts`) under service-role and persists via `practice_generate_self_test`, **idempotent on `tests.client_request_id`** so a reclaimed/re-run job can't duplicate a test or double-charge. Survives the 300s function ceiling and client disconnects. Flag off ⇒ streaming path (the default today).
- **Worker** (`app/api/internal/practice/run-jobs`): claims `grade|pdf|email|tracker_update|assign_generate_test|review_generate|student_generate_test` via `practice_claim_jobs` with per-type timeouts; grading reclaim leases off `claimed_at`, completion fenced on `claimed_by` (no concurrent double-grade).
- **Closed learning loop (spaced-repetition review):** pure SM-2-lite scheduler `review-schedule.ts` (pass ≥75%, ease 1.3–2.6, graduates after 3 consecutive passes) advanced on every grade via extra keys in the `practice_update_trackers_bulk` payload (`review-schedule-payload.ts`). Nightly scheduler (`/api/internal/practice/review-scheduler`, pg_cron 02:30 UTC) selects due trackers under pure caps (`review-selection.ts`: ≤8 review tests/billing period, ≤1/day, quota-aware) and a staged-rollout cohort gate (`review-cohort.ts`: FNV-1a percentage bucket + org allowlist; defaults to nobody), enqueues `review_generate` jobs → `materializeReviewPracticeTest` (`review-generation.ts`) biases the pipeline to the due weak topic, persists via `practice_generate_review_test` (`test_type='review'`), and sends a `review_ready` notification. Global kill switch `review_scheduler_enabled()`. Advisory ranker `src/lib/student/review-advisory.ts` (overdue → due-soon → weak, cap 5) powers "what to focus on next" surfaces (student/parent/teacher).
- **Structured output** (`src/lib/ai/structured-output.ts`): OpenAI strict structured output vs DeepSeek JSON-mode bifurcation.
- **Grading** (`ai-grade-practice-test.tsx` + `grading-*.ts`): per-answer scoring + model/user answer summaries + feedback → `student_answers` + `test_reports` → `performance_tracker` rollup (now including the review-schedule advance).
- **Teacher class insight** (`src/lib/teachers/teacher-class-insight*.ts`): AI dashboard narrative + at-risk intervention suggestions, cached in `teacher_class_insights` keyed by data fingerprint (see [teacher-portal.md](teacher-portal.md)).
- **Known reliability gaps:** no independent answer-key verifier, distractor judge, self-consistency ensemble, VLM diagram judge, or golden-set CI regression harness (see [student-portal.md](student-portal.md) §8.5).

## 5. Doubt-chat (AI tutor) — [Technical]
Modes `explain | solve_with_me | quiz_me` (`doubt-tutor-mode.ts`). On-syllabus scope (`scope-precheck.ts`, `validate-doubt-scope.ts`); 12 subject packs (`docs/doubt-subject-packs/*.md`) + shared preamble; deterministic safety screen (`safety.ts`, `safety-detectors.ts`); attachments (image/PDF) → Storage with OCR `ocrText`; KaTeX + Markdown rendering (display-math indentation preserved inside lists; chat skips the practice-autofix whitespace span merge). Gated by `canStartDoubtChat`; output tokens metered via `consumeTokens` — conversation ownership is verified **before** debiting, and adaptive follow-ups (`appendAdaptiveFollowups`) are entitlement-gated, token-debited, and bounded by a 60s abort timeout.
- **Mistake-grounded chats:** "Ask about this" on a graded wrong question passes `mistakeContext.questionId` (`request-schema.ts`); `loadMistakeForQuestion` (`src/lib/doubt/mistake-context.ts`) ownership-checks the question (student's own test, incorrect answer only) and injects question / student answer / reference answer / grader feedback into the prompt scope.
- **Safety-screen hardening:** multi-category turns store one representative `moderation_flags.source` (highest severity) with the full breakdown in `reason` (no varchar(30) overflow silently dropping flags); sexual-violence terms moved to a review-only "sensitive topic" tier so curricular questions are answered, not blocked; admin `content_blacklist` patterns are ReDoS-guarded (length/nesting refusal + per-turn scan cap); blocked-turn moderation-flag + analytics writes are awaited before the 422; message array capped at 1000.

## 6. Billing & entitlements — [Plain] + [Technical]
- **[Plain]** Three plans: **Free Trial** (₹0, 5 tests, 50k AI tokens), **Pro Monthly** (₹600, 30 tests, 200k/400k tokens by grade), **Pro Annual** (₹6,000 — 2 months free, ~17% saving, effective ₹500/mo; 360 tests, 2.4M/4.8M tokens by grade). Access is gated by two meters: practice **tests** and doubt-chat **AI tokens**, reset each billing period. (Catalog: `src/lib/billing/plans.ts`, seeded via `20260531120000_update_student_plan_prices.sql`.)
- **[Technical]** `plans`, `subscriptions` (status `trialing|active|coupon|grace|past_due|cancelled|expired`; `paused` can arrive from Razorpay and is treated as **no access** regardless of remaining quota), `usage_periods` (tests/tokens quota+used). Entitlement snapshot (`src/lib/billing/entitlements.ts`) derives `canStartTest` (testsLeft>0, OR an available manual admin grant via `hasAvailableQuotaTestGrant` when the period quota is exhausted) and `canChatDoubt` (tokensLeft>0). **Fail-closed** on billing infra errors; `consumeTest`/`refundTest` are compensating; `SAAS_ENFORCEMENT=false` or `staffOverride` bypass. One trial per normalized identity (`free_trial_claims`). Razorpay webhooks idempotent via `billing_events.razorpay_event_id` + `UNIQUE(razorpay_subscription_id)`; `subscription.paused/resumed` zero/restore `usage_periods` quota idempotently (`pre_pause_quota` JSONB validated on read); `subscription.charged` derives period end from the plan interval (no zero-length periods); receipt/activation emails carry a stable `dedupKey` so webhook replays don't re-send. Change-plan/create-subscription take an atomic `pending_plan_code` claim before calling Razorpay (double-submit can't create two subscriptions). Dunning runs off the dedicated `dunning_started_at` clock; reconciliation auto-heals the revenue-leak direction (risky auto-grants stay log-only) — all under `app/api/internal/billing/**` + admin tools.

## 7. Notifications & email — [Technical]
In-app `notifications` + Resend email via `src/lib/notifications/**`: report-ready, usage thresholds (80/100%, idempotent via `usage_notification_log`), trial reminders, assignment events (`assignment_graded` is dedup-indexed so re-grades stay idempotent), **`review_ready`** (spaced-repetition review test materialized; "Start review" CTA → `/student/practice/<testId>`), account-security, org events. List queries are backed by `(recipient, created_at)` + unread partial indexes (`20260703000000`). Admin broadcasts/templates with delivery webhooks (`comms_audit`, `email_webhook_events`); the Resend webhook authenticates via Svix signature, with a dedicated `RESEND_WEBHOOK_BEARER` fallback that is closed when unset. Per-portal notification APIs are rate-limited.

## 8. Security & compliance — [Technical]
- CSRF/origin gates (§2), Postgres-backed rate limiting with a shared **fail-closed-in-prod** degraded policy (`src/lib/ratelimit/fail-policy.ts`, adopted by auth/contact/feedback/teacher/parent limiters; admin-login limiter logs loudly on DB error), CSP with per-request nonce (`src/lib/security/csp.ts`), input validation (Zod; auth schemas `.strict()`), server-side sanitizer for rich practice answers at the single write choke point (`rich-answer-sanitize-server.ts`, shares the editor allowlist).
- AI-output moderation (`src/lib/ai/moderation.ts`, `moderation_flags`, `content_blacklist` — ReDoS-guarded, §5).
- **Function-level authz on SECURITY DEFINER RPCs** (§3) closed the IDOR class where `authenticated`-granted RPCs trusted caller-supplied ids; parent-link student confirmation is restored and trigger-enforced; admin RPCs are postgres-only with GUC checks. Route-level closures: student `checkout-event` requires `role == student`; parent `open-report` verifies the test belongs to the linked student.
- **CI guard-tests:** every student/parent/teacher API route must authenticate and every admin route must call `requireAdminApi` (added in `accc7c8`/`ac2df61`); plus Drizzle↔Postgres parity (blocking), FK-parity script (`pnpm db:check-fk-parity`), migration CONCURRENTLY lint.
- Sentry PII scrubbing covers events and the Logs stream in all three runtimes (`before-send.ts`).
- DPDP-style: `parental_consents` (guardian consent for minors), `compliance_requests` (DSR: `open → in_progress → fulfilled|rejected|erased`, statutory `dueAt`), `retention_policies`, anonymisation/export/erasure. Erasure (`src/lib/compliance/erasure.ts`) now also deletes behavioral/engagement tables (`practice_analytics_events`, `practice_generation_runs`, `practice_jobs`, `student_activity_streaks`, `teacher_student_links`), with a drift-guard test forcing every student-identified table to be classified; admin hard-delete = erase + auth ban (the auth row is retained because `audit_logs`/`payments`/`tests` keep NO ACTION FKs).
- Audit: append-only `admin_action_log`; `writeAdminActionStrict` makes the audit write load-bearing for staff-override/impersonation/hard-delete; rich linking audit in the parent flow.
